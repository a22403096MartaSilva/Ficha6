1. Arquitetura MVT do Django

O Django utiliza a arquitetura MVT (Model, View, Template), que separa a aplicação em três partes: dados, lógica e apresentação.

O Model representa os dados da aplicação. No meu projeto, os models estão no ficheiro models.py e incluem entidades como Projeto, Tecnologia, Competência, Formação e Making Of. Estes models definem como os dados são guardados na base de dados.

A View trata da lógica da aplicação. No meu projeto, as views estão no ficheiro views.py. Por exemplo, a index_view mostra a lista de projetos e a projeto_view permite criar um novo projeto através de um formulário.

O Template é responsável pela apresentação visual. São ficheiros HTML que mostram os dados ao utilizador, como index.html, pagina_projeto.html e sobre.html.

O fluxo de informação funciona da seguinte forma: o utilizador acede a um URL no browser, o Django procura essa URL no ficheiro urls.py, chama a view correspondente, a view pode consultar ou guardar dados através dos models, e depois devolve um template HTML ao utilizador.

Por exemplo, quando o utilizador acede à página inicial, o Django chama a index_view. Esta view vai buscar os projetos à base de dados com Projeto.objects.all() e envia-os para o template index.html.

Outro exemplo é a criação de um projeto. Quando o utilizador acede a /projeto/, a view projeto_view apresenta um formulário. Quando o formulário é submetido, os dados são validados e guardados na base de dados.

2. Modelação

A modelação da aplicação foi feita no ficheiro models.py. Cada classe representa uma tabela na base de dados e cada atributo representa um campo.

No meu projeto foram criadas as seguintes entidades: Projeto, Tecnologia, Competência, Formação, MakingOf, EmpresaParceira, Docente, Licenciatura, TFC e UnidadeCurricular. Para esta website foram utilizadas as 5 primeiras.

Existem relações entre estas entidades. Por exemplo, um Projeto pode ter várias Tecnologias e várias Competências. Uma Formação também pode estar associada a várias Competências.
